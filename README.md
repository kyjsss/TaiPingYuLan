## 系统概述

本系统是一次基于古籍原生分类体例的知识图谱重构实验。本项目放弃了通用大模型对非结构化文本的机械实体抽取，转而复用《太平御览》自带的“部—门—子目”层级分类直接映射为图谱底层架构，并在不同部类节点间进行了受控的横向关联匹配。

项目旨在验证：将传统学术遗产转化为引导现代语用模型推演的受控框架，能否在控制算力成本的前提下，有效抑制大模型的“幻觉”并提供具有溯源保障的文史辅助检索机制。

### 核心特性

- 🎯 **双模路由**：提供“以点带面的旁证溯源机制”与“由宏观到微观的倒置提取路径”。
- 📊 **原生知识图谱**：直接映射古籍编排逻辑建立“社区即实体”的检索底座。
- 🔍 **复合增强**：图谱连线结合 FAISS 局部向量相似度计算。
- 🤖 **受控文案生成**：强制附带出处与检索逻辑标签作为背景输入，收拢 LLM 推演边界。

## 快速开始

### 环境要求

- Python >= 3.8
- 依赖包见 [`requirements.txt`](requirements.txt)

### 安装依赖

```bash
pip install -r requirements.txt
```

### 基础配置

需要提供大语言模型 API 密钥（默认配置下层调用的是 SiliconFlow 提供的接口）。
编辑 [`config.py`](config.py) 或通过命令行设置环境变量：

```bash
export SILICONFLOW_API_KEY="您的_API_KEY"
```

## 系统参数说明

为了支持学术验证与系统效能的对比分析，本原型系统在 `config.py` 文件中暴露了大量可调优的核心参数。
因篇幅所限，此处仅做分类概述。详细说明与默认值设定，请直接查阅源码 [config.py](config.py)。

1. **API与模型配置**: 包含鉴权密钥、所用的大语言模型（默认 DeepSeek-V3.2）、嵌入向量模型（默认 Qwen3-Embedding）及其各类生成阈值。
2. **检索基本过滤参数**: 控制向量相似度的最初通行门槛 (`similiarity_threshold`)，以及全局最大文本吞吐预算 (`global_char_budget`)。
3. **图谱配额与路由参数 (Micro/Macro)**: 控制两种提取机制的扇出广度。比如决定周边节点提取比例的 `neighbor_quota_ratio`，以及单点向外延伸的最远链路数 `neighbor_fanout` 等。
4. **长文处理机制**: 提供滑动切片并控制强制上下文段落保留的界限 `long_text_threshold`。

通过细致微调上述杠杆，系统能够在“紧贴原生条目的高精确度还原”与“强推演性的广域关联综述”两种风格间进行平滑过渡。

## 执行命令指南

主程序 `main.py` 封装了所有模块的调用链路，其命令行设计支持从基础路由到极其精细的执行参数微调。全部可调标志请参阅 `python main.py --help`。

### 核心必需参数
- `QUERY` (字符串): 最后输入的必填项，即您要提问或查验的内容。

### 1. 基础模式切换 (`--mode`)
支持论文中论述的两种提取路径检验：

*   **以点带面的旁证溯源机制 (默认 Micro 模式)**
    此模式从具体节点辐射，适合进行微观考据：
    ```bash
    python main.py "荧惑守心有何预兆？"
    ```

*   **由宏观到微观的倒置提取路径 (Macro 模式)**
    此模式跨越高层提取综述，适合讨论长时段或系统层级的问题：
    ```bash
    python main.py --mode macro "天人感应的原理"
    ```

### 2. 执行过程观测与格式化 (`--verbose`, `--output`)

*   **查看系统调度链路日志**：
    附带 `--verbose` 将打印出系统底层的详尽决策链，包括意图转译、锚点选择、算力配额划分与关联链路追踪（极其推荐在审查逻辑时开启此项）：
    ```bash
    python main.py --verbose "古人如何应对旱灾？"
    ```

*   **调整输出载体格式**：
    使用 `--output [text|json|markdown]` 指定生成的最终排版：
    ```bash
    python main.py --output json "五行相生相克"
    ```

### 3. 微观调整调度参数（针对 Micro 模式的即席干预）

您可直接在命令行覆盖 `config.py` 中的阈值，以快速验证调优效果（例）：

*   **限制关联发散**（切断图谱衍生，回归传统只看向量的 RAG）：
    ```bash
    python main.py --graph-enable false "日食相关的怪异记录"
    ```

*   **强化旁证与衍生抓取**（比如大幅提供邻居社区的发散配额，将单节点的扇出开到 10，适合极其生僻的长尾名词延展）：
    ```bash
    python main.py --neighbor-quota-ratio 0.6 --neighbor-fanout 10 "九服之制"
    ```

### 4. 其它功能项

*   **忽略意图转化缓存**：
    每次问题提取都会通过大模型转译以对齐古典语境，默认开启缓存。通过以下参数可强制每次重新生成：
    ```bash
    python main.py --no-cache "通货膨胀"
    ```

## 核心模块指引

系统结构清晰，由以下主要逻辑串联：

- **`data_manager.py` (数据管理)**：承当底层，负责载入已被树状映射和补全后的原生材料矩阵，并初始化快速近似检索模块（FAISS）。
- **`query_processor.py` (查询处理)**：负责第一道关卡，将抽象的现代查询提问意图利用 LLM 转译为类书古典文本语境下的检索术语，完成古今语义对齐。
- **`retriever.py` (路由与检索)**：核心分发，承载两种图谱游走抽取策略的实现，是体现定制检索路径最主要的位置。
- **`processor.py` 和 `fusion.py` (文本重组与合成)**：前者负责执行长文首尾留存与有效切片；后者整合离散条目并作为最后动作拦截口——给将要输送给生成模型的每条语料，预先强制打上前置的逻辑构建与出处标签，保障最终推演结果的可溯源性。
